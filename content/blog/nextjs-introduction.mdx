# Introduzione a Next.js: Una Guida per Iniziare

Next.js è un framework open-source basato su React che rende lo sviluppo di applicazioni web moderne più semplice e performante. Creato dal team di Vercel, Next.js è progettato per combinare le migliori pratiche dello sviluppo front-end con funzionalità avanzate come il rendering lato server (SSR), il rendering statico (SSG) e l'ottimizzazione automatica delle performance.

In questa guida, esploreremo cosa rende Next.js speciale e ti mostreremo come iniziare a costruire la tua prima applicazione.

---

## Perché scegliere Next.js?

### 1. **Rendering flessibile**
Next.js supporta diverse modalità di rendering:
- **Server-Side Rendering (SSR):** Il contenuto è generato sul server ad ogni richiesta.
- **Static Site Generation (SSG):** Le pagine vengono generate in anticipo durante il build e servite come file statici.
- **Incremental Static Regeneration (ISR):** Permette di aggiornare i contenuti statici senza ricostruire l'intera applicazione.

### 2. **Routing integrato**
Next.js utilizza un sistema di routing basato sui file: ogni file creato nella cartella `pages/` corrisponde a una route.

### 3. **API Routes**
Puoi creare API direttamente all'interno del progetto utilizzando la cartella `pages/api/`.

### 4. **Ottimizzazione delle performance**
Con funzionalità come il code splitting, il prefetching delle pagine e il supporto a immagini ottimizzate, Next.js è progettato per rendere le applicazioni veloci e scalabili.

## Prerequisiti
Prima di iniziare, assicurati di avere:
- **Node.js** (versione 16 o successiva) installato.
- Un editor di codice, come Visual Studio Code.
- Familiarità con React e JavaScript.

---

## Creare un nuovo progetto Next.js

### Passo 1: Installazione
Esegui il comando seguente per creare un nuovo progetto:
```bash
npx create-next-app@latest my-next-app
```
Sostituisci `my-next-app` con il nome del tuo progetto. Ti verrà chiesto di configurare alcune opzioni, come l'uso di TypeScript e di ESLint.

### Passo 2: Avviare l'applicazione
Accedi alla directory del progetto e avvia il server di sviluppo:
```bash
cd my-next-app
npm run dev
```
Il server sarà disponibile su `http://localhost:3000`.

---

## Struttura del progetto
Ecco una panoramica della struttura del progetto:

```plaintext
my-next-app/
├── pages/
│   ├── index.js       // Homepage
│   ├── about.js       // Esempio di pagina aggiuntiva
│   └── api/
│       └── hello.js   // Esempio di API route
├── public/            // File statici (immagini, favicon, ecc.)
├── styles/            // File CSS
├── next.config.js     // Configurazioni personalizzate
└── package.json       // Dipendenze del progetto
```

### Creare una nuova pagina
Per aggiungere una nuova pagina, crea un file all'interno della cartella `pages/`. Ad esempio, per una pagina "Chi siamo":

```javascript
// pages/about.js
export default function About() {
  return (
    <div>
      <h1>Chi siamo</h1>
      <p>Benvenuto nella pagina Chi siamo!</p>
    </div>
  );
}
```

Visita `http://localhost:3000/about` per visualizzare la nuova pagina.

## Styling delle pagine
Next.js supporta diverse opzioni per lo styling:

### CSS Modules
I file CSS possono essere utilizzati come moduli per evitare conflitti di classi:

```css
/* styles/Home.module.css */
.title {
  color: blue;
  font-size: 2rem;
}
```

```javascript
// pages/index.js
import styles from '../styles/Home.module.css';

export default function Home() {
  return (
    <h1 className={styles.title}>Benvenuto in Next.js!</h1>
  );
}
```

### Styled Components
Puoi utilizzare librerie come Styled Components per uno styling basato su componenti:

```bash
npm install styled-components
```

```javascript
import styled from 'styled-components';

const Title = styled.h1`
  color: blue;
  font-size: 2rem;
`;

export default function Home() {
  return <Title>Benvenuto in Next.js!</Title>;
}
```

## Configurazione delle API Routes
Crea API direttamente nella cartella `pages/api/`. Ad esempio, per un'API che restituisce un messaggio di benvenuto:

```javascript
// pages/api/hello.js
export default function handler(req, res) {
  res.status(200).json({ message: 'Ciao da Next.js!' });
}
```

Visita `http://localhost:3000/api/hello` per testarla.

---

## Deploy della tua applicazione
Quando sei pronto per il deploy, utilizza il comando:
```bash
npm run build
```

Questo genererà una build ottimizzata nella cartella `.next/`. Puoi quindi utilizzare piattaforme come [Vercel](https://vercel.com/) o altri hosting provider per pubblicare la tua applicazione.
